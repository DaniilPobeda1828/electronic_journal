## **4. Выявленные проблемы и их решение**

### **4.1. Ошибка: `UNIQUE constraint failed: users.login`**
```
sqlalchemy.exc.IntegrityError: (sqlite3.IntegrityError) UNIQUE constraint failed: users.login
```
**Причина:** Попытка повторного создания пользователя с уже существующим логином.  
**Решение:** Добавлена проверка существования пользователя перед созданием через `User.query.filter_by(login=...).first()`.

### **4.2. Ошибка: `NOT NULL constraint failed: users.password_hash`**
```
sqlite3.IntegrityError: NOT NULL constraint failed: users.password_hash
```
**Причина:** Создание пользователя без вызова `set_password()`.  
**Решение:** Добавлен вызов `user.set_password()` для всех создаваемых пользователей:

```python
# Было (неправильно)
admin = User(login='admin_test_model', full_name='Admin', role='admin')

# Стало (правильно)
admin = User(login='admin_test_model', full_name='Admin', role='admin')
admin.set_password('admin_pass')
```

### **4.3. Ошибка: `NOT NULL constraint failed: subjects.teacher_id`**
```
sqlite3.IntegrityError: NOT NULL constraint failed: subjects.teacher_id
```
**Причина:** При создании предмета `teacher_id` передавался как `None`.  
**Решение:** Исправлен порядок сохранения объектов — сначала создаётся преподаватель, затем предмет с его `id`.

```python
# Правильный порядок:
teacher = User(login='schedule_teacher', full_name='Schedule Teacher', role='teacher')
teacher.set_password('pass')
db.session.add(teacher)
db.session.flush()  # Получаем teacher.id

subject = Subject(name='Schedule Subject', teacher_id=teacher.id)
db.session.add(subject)
```

### **4.4. Ошибка: `DetachedInstanceError`**
```
sqlalchemy.orm.exc.DetachedInstanceError: Instance <User at ...> is not bound to a Session
```
**Причина:** Использование объектов вне контекста сессии SQLAlchemy.  
**Решение:** Передача `id` объектов вместо самих объектов в фикстуре `init_database`:

```python
# Возвращаем ID, а не объекты
return {
    'admin_id': admin.id,
    'teacher_id': teacher.id,
    'student_id': student.id,
    'group_id': group.id,
    'subject_id': subject.id
}
```

### **4.5. Предупреждения: `LegacyAPIWarning`**
```
LegacyAPIWarning: The Query.get() method is considered legacy as of the 1.x series
```
**Причина:** Использование устаревшего метода `Query.get()`.  
**Решение:** Замена на `db.session.get(Model, id)` в будущих версиях (на работу тестов не влияет).
