# Техническое задание
## Разработка системы учёта успеваемости студентов (электронный журнал)

### 1. Что разработать
Программу для учёта успеваемости студентов — электронный журнал для преподавателя и заведующего отделением.

---

### 2. Технологии

| Компонент | Технология |
|-----------|------------|
| Язык | Python |
| Интерфейс | Tkinter / PyQt / Flask (веб) |
| База данных | SQLite |
| Сборка | PyInstaller (.exe) |

---

### 3. Функциональные требования

#### 3.1. Управление группами и студентами
- Добавление/удаление групп (например, ИС-21, ПРО-23, РПО-24).
- Добавление/удаление студентов (ФИО, группа, дата рождения).
- Просмотр списка групп и студентов в группе.

#### 3.2. Управление предметами и оценками
- Добавление/удаление предметов (ОП, Базы данных, Веб-программирование).
- Выставление оценки студенту по предмету с указанием:
  - даты
  - оценки (2, 3, 4, 5)
  - типа работы (лекция / практика / экзамен)
- Автоматический подсчёт **среднего балла** по каждому предмету.

#### 3.3. Отчёты и статистика
| Отчёт | Что показывает |
|-------|----------------|
| Успеваемость группы | Средний балл всех студентов группы |
| Должники | Студенты со средним баллом < 3 |
| Отличники | Студенты со средним баллом ≥ 4.75 |
| Сравнение групп | Таблица успеваемости между группами |

#### 3.4. Журнал действий (логи)
- Запись всех действий:
  - Кто поставил оценку, когда, какую
  - Кто добавил/удалил студента или группу
  - Кто добавил/удалил предмет
- Просмотр журнала с фильтром по дате и пользователю.

#### 3.5. Роли и права доступа

| Роль | Возможности |
|------|-------------|
| **Студент** | Только просмотр своих оценок |
| **Преподаватель** | Выставление оценок по своим предметам, просмотр успеваемости своих групп |
| **Зав. отделением (админ)** | Всё: управление группами, предметами, студентами, пользователями |

---

### 4. Что замерить (характеристики ПО)

| Параметр | Способ замера |
|----------|---------------|
| Время запуска программы | Секундомер (от клика до интерфейса) |
| Время открытия журнала группы | Секундомер |
| Время расчёта среднего балла по всем группам | Секундомер |
| Потребление оперативной памяти | Диспетчер задач / `task manager` |

*Результаты замеров занести в отчёт по практике.*

---

### 5. Что сделать для защиты (программными средствами)

- Авторизация по логину/паролю.
- Пароли хранить в **хэшированном** виде (hashlib / bcrypt).
- Студент не может видеть чужие оценки.
- Преподаватель не может менять оценки других преподавателей.
- Логирование неудачных попыток входа (IP, время, логин).
